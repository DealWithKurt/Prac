Проект тестирования «Мой Фитнес»
Чеклист: подключение источников активности, выгрузка и учёт активности
1. Подключение источников активности
•	Приложение запрашивает разрешение на доступ к данным о активности (для Android / iOS)
•	Доступны основные источники: шагомер, GPS (бег/ходьба), сторонние приложения (Google Fit, Apple Health, Garmin, Polar, Strava — согласно документации)
•	При выборе источника (например, Google Fit) выполняется редирект на авторизацию в сервисе
•	После успешной авторизации источник отображается в списке подключённых
•	Можно отключить источник: данные перестают синхронизироваться
•	При отключении источника появляется подтверждение
•	При повторном подключении того же источника данные за период отключения не дублируются
•	Если источник требует обновления токена — приложение запрашивает повторную авторизацию
2. Выгрузка активности
•	После подключения источника приложение автоматически выгружает активность за последние N дней (по документации)
•	При ручном обновлении (pull-to-refresh или кнопка «Синхронизировать») запускается выгрузка новых данных
•	Во время выгрузки отображается индикатор загрузки
•	При отсутствии интернета выгрузка не стартует, показывается сообщение
•	После восстановления интернета выгрузка автоматически продолжается (или требуется ручной повтор)
•	Выгруженные активности отображаются в ленте / календаре тренировок с корректными временем, типом, дистанцией, калориями
•	Повторная выгрузка той же активности не создаёт дубликат
•	При ошибке выгрузки (например, 500 от сервера) появляется понятное сообщение и кнопка повтора
3. Учёт активности (подсчёт статистики)
•	После выгрузки обновляются шаги за сегодня (если источник — шагомер)
•	Обновляются потраченные калории (дневные / за тренировку)
•	В профиле отображается общая статистика: шаги за неделю, километры, активные минуты
•	При добавлении ручной тренировки (если функция доступна) она суммируется с данными из источников без задвоения
•	При удалении источника его данные перестают учитываться в статистике (но исторические записи могут оставаться — по спецификации)
•	Данные учёта корректно отображаются в виджетах геймификации (очки, уровни, ачивки)

Тест-кейсы: функционал авторизации / регистрации
Предусловие: приложение «Мой Фитнес» установлено, пользователь не авторизован.
TC-01 – Регистрация нового пользователя по email
Шаги:
1.	Открыть приложение
2.	Нажать «Регистрация»
3.	Ввести валидный email (например, test@example.com)
4.	Ввести пароль (минимум 6 символов, цифра/буква — согласно требованиям)
5.	Подтвердить пароль
6.	Нажать «Зарегистрироваться»
ОР:
Появляется сообщение об успешной регистрации. Пользователь автоматически авторизован и попадает на главный экран приложения. В профиле отображается введённый email.
TC-02 – Регистрация с уже существующим email
Шаги:
1.	Повторить TC-01 с email, который уже зарегистрирован в системе
ОР:
Появляется ошибка: «Пользователь с таким email уже существует». Регистрация не выполняется.
TC-03 – Слабый пароль при регистрации
Шаги:
1.	Начать регистрацию
2.	Ввести email
3.	Ввести пароль из одних цифр (например, 123456)
4.	Подтвердить его
ОР:
Появляется предупреждение «Пароль должен содержать буквы и цифры» (или другое правило из документации). Кнопка регистрации неактивна или выдаёт ошибку.
TC-04 – Вход с валидными учётными данными
Шаги:
1.	На главном экране нажать «Вход» / «У меня есть аккаунт»
2.	Ввести зарегистрированный email и пароль
3.	Нажать «Войти»
ОР:
Успешный вход, открывается главный экран приложения.
TC-05 – Вход с неверным паролем
Шаги:
1.	Ввести корректный email, неверный пароль
2.	Нажать «Войти»
ОР:
Ошибка: «Неверный email или пароль». Пользователь остаётся на экране входа.
TC-06 – Восстановление пароля
Шаги:
1.	На экране входа нажать «Забыли пароль?»
2.	Ввести email, на который зарегистрирован аккаунт
3.	Нажать «Отправить ссылку для сброса»
ОР:
Сообщение: «Ссылка для сброса пароля отправлена на ваш email». Письмо приходит в течение 2 минут (проверяется в тестовом почтовом ящике).
TC-07 – Выход из аккаунта
Шаги:
1.	Авторизоваться
2.	Перейти в профиль / настройки
3.	Нажать «Выйти»
4.	Подтвердить выход (если есть диалог)
ОР:
Пользователь возвращается на экран входа/регистрации. При повторном открытии приложения сессия не восстанавливается.
TC-08 – Оставаться в системе (запоминание сессии)
Шаги:
1.	Авторизоваться (без отметки «Выйти»)
2.	Полностью закрыть приложение
3.	Открыть снова
ОР:
Пользователь остаётся авторизованным, не требуется повторный ввод пароля.
